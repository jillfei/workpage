🎯 1. 需求背景与目标
  • 背景：当前用户在新增/编辑智能体时，编写高质量指令（Prompt）学习成本高、耗时长。现有"AI 优化"功能仅支持对已有文本进行润色，缺乏快速复用标准化/优质提示词的能力。
  • 目标：在指令模块新增 提示词库 入口，提供模板检索与一键应用功能，降低 Prompt 编写门槛，提升智能体配置效率与一致性。
  • 适用范围：智能体「新增页」与「编辑页」的指令（System Prompt / Instruction）模块。
📦 2. 功能概述

在指令输入区域操作栏右侧，于 AI 优化 按钮左侧新增 [提示词库] 按钮。点击后唤起模板选择弹窗，支持名称搜索、列表展示、分页排序；点击"应用模板"可将内容注入指令框，并包含内容覆盖二次确认机制。

🛠 3. 详细功能需求
3.1 入口与 UI 布局
元素位置交互/样式说明
[提示词库] 按钮指令模块操作区右侧，紧贴 AI 优化 按钮左侧标准按钮样式，Hover 状态反馈，点击唤起 Modal
3.2 提示词库弹窗（核心交互）

3.2.1 搜索区

  • 控件：输入框（输入内容实时触发）
  • 逻辑：支持按 模板名称 进行模糊搜索；清空输入框自动恢复默认列表。

3.2.2 列表展示区

字段展示规则说明
模板名称完整展示支持 Tooltip 悬浮显示全称（若被截断）
摘要/Prompt内容截断展示（建议：50 字符 + ...）鼠标 Hover 可显示完整前段内容
创建人展示姓名-
更新时间YYYY-MM-DD HH:mm默认倒序排序依据
操作列[应用模板] 按钮点击触发应用逻辑

3.2.3 分页与排序

  • 数据来源：提示词模块，状态为启用的所有提示词模板。
  • 排序规则：固定按 更新时间 倒序排列（最新在前）。
  • 分页规则：每页固定 10 条。
    • 总数据量 ≤ 10：隐藏分页组件。
    • 总数据量 > 10：显示标准分页器，支持页码跳转与前后翻页。
3.3 交互与业务逻辑

3.3.1 应用模板逻辑

  1. 用户点击某条模板的 [应用模板] 按钮。
  2. 系统校验智能体指令输入框当前状态：
    • 若为空：直接将该模板完整 Prompt 内容写入输入框，自动关闭弹窗，光标定位至输入框末尾。
    • 若已有内容：弹出二次确认 Dialog：
      提示："应用新模板将覆盖当前指令内容，是否继续？"
      按钮：取消 | 确认
      • 点击 取消：关闭确认框，保留弹窗不关闭，指令框内容不变。
      • 点击 确认：执行覆盖写入，关闭弹窗，光标定位至末尾。
  3. 写入后，用户可继续在输入框内进行手动增删改，不受限制。
⚠️ 4. 异常状态与边界处理
场景处理策略
模板列表请求失败/网络异常弹窗内展示全局错误提示（如："加载失败，请重试"），提供刷新按钮
搜索无结果展示空状态插画 + 文案"未找到匹配的提示词模板，请调整搜索词"
应用后超出输入框字符上限截断至最大允许长度，并 Toast 提示"内容已应用，超出限制部分已自动截断"
