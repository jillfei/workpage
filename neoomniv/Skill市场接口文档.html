一、接口认证要求

讯飞 SkillHub 的公开技能（visibility=PUBLIC）可以通过 /api/web/ 前缀的接口完全匿名访问，无需注册和登录。

能力是否可用认证要求
技能列表（分页）✅ 可用无需认证
技能详情✅ 可用无需认证
版本列表✅ 可用无需认证
技能分类✅ 可用无需认证
文件列表✅ 可用无需认证
单文件下载✅ 可用无需认证
关键词搜索✅ 可用无需认证
收藏/评分❌ 需认证需要登录
发布技能❌ 需认证需要登录

当前公开技能总数：2726 个

二、完整接口文档

1. 获取技能列表（分页 + 搜索 + 分类 + 排序）

GET /api/web/skills?q={q}&namespace={namespace}&label={label}&sort={sort}&page={page}&size={size}

查询参数（全部可选）：

参数类型说明示例
qstring关键词搜索（最少2字符，截断至50字符）python, 翻译, AI
namespacestring按命名空间过滤（@前缀会自动去除）global, astroclaw
labelstring按分类 slug 过滤（见下方分类列表）coding, productivity
sortstring排序方式（见下方排序列表）relevance, downloads
pageint页码（从0开始）0
sizeint每页数量（如需直接获取全量数据，写10000）10, 20, 100

curl "https://skill.xfyun.cn/api/web/skills?q=支付&namespace=astroclaw&sort=downloads&page=0&size=10"

2. 获取分类列表

GET https://skill.xfyun.cn/api/web/labels

无需认证。 返回所有可用的技能分类。

3. 排序参数

排序（sort）可选值（代码配置）：

值说明备注
relevance相关性排序（默认）有 q 参数时按搜索相关性；无 q 时按平台默认排序
downloads按下载量服务端可能结合其他因子做复合排序
newest按最新同上
注意：排序参数均被服务端接受（返回 code=0），但实际排序效果可能受服务端复合排序逻辑影响。不支持自定义排序方向（如 desc/asc），服务端自行决定排序方向。

排序验证示例：

# coding 分类 + 按下载量排序
curl "https://skill.xfyun.cn/api/web/skills?label=coding&sort=downloads&page=0&size=10"

7. 下载单个文件

GET /api/web/skills/{namespace}/{slug}/versions/{version}/file?path={filePath}

无需认证。 直接返回文件内容。

请求示例：

# 下载 SKILL.md（技能定义文件）
curl "https://skill.xfyun.cn/api/web/skills/global/cet-exam-trainer/versions/20260519.134423/file?path=SKILL.md"

# 下载参考资料
curl "https://skill.xfyun.cn/api/web/skills/global/cet-exam-trainer/versions/20260519.134423/file?path=references/四六级核心词汇.md"

# 下载元数据
curl "https://skill.xfyun.cn/api/web/skills/global/cet-exam-trainer/versions/20260519.134423/file?path=_meta.json"

SKILL.md 文件格式：

---
name: cet-exam-trainer
description: 专业四六级英语备考助手...
---

三、接入方案

方案：实时从讯飞拉取数据

用户搜索请求
  → 代理到 /api/web/skills?q=xxx&label=xxx&sort=xxx&page=xxx
  → 返回结果
  → 用户点击技能
  → 代理获取详情和文件内容

优点：

  • 数据实时，无需存储

缺点：

  • 依赖外部接口可用性和速度
  • 讯飞可能随时变更接口
  • 限流未明确
